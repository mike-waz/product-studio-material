## Working on Project Studio Itself

These are for when you're developing/improving Project Studio as a tool.

### Status & Memory

| To do this... | Say... |
|---------------|--------|

| Wrap up a session | "Let's wrap up" or "Capture where we are" |

## Test Mode (Currently ON by default)

Test mode generates structured feedback to improve Project Studio.

| To do this... | Say... |
|---------------|--------|
| End test and get report | "End test" or "Finish test" or "Generate test report" |
| Skip tester feedback | "Skip my feedback" (Claude will still self-evaluate) |
| Turn off test mode | Edit CLAUDE.md Test Mode section |

**What happens at end of test:**
1. Claude evaluates: Clarity, Workflow, Output Quality, Speed
2. Claude asks you 4 feedback questions
3. Report saved to `tests/[test-name]-[date].md`
4. Summary provided to bring back to dev session

## Tips

- **Be specific** ‚Äî "Create a student list with search, filters, and a table" works better than "Create a list"
- **One thing at a time** ‚Äî Claude builds incrementally, so you'll see progress after each piece
- **Review often** ‚Äî Catch issues early by checking after each addition
- **Wrap up sessions** ‚Äî Say "let's wrap up" to ensure status is captured before ending
